# 文档类型与章节最佳实践总表

本文档汇总各类型文档的目的、读者、必备/可选章节、禁忌与质量标准，便于 Skill 或人工生成时查阅。细节以各 `*-guide.md` 为准。

---

## 总表

| 文档类型 | 目的 | 主要读者 | 必备章节（摘要） | 可选章节 | 禁忌 | 质量判断 |
|----------|------|----------|------------------|----------|------|----------|
| README | 项目入口、快速开始 | 所有人 | 名称与描述、功能、快速开始、文档索引、版本与许可 | 架构概览、技术栈、FAQ、贡献 | 与 Memory 矛盾、步骤不可跑通、无文档索引 | 5 分钟可理解并跑通 |
| 项目概述 | 目标、范围、成功准则 | 产品、PM、架构 | 目标、范围、成功准则、约束 | 路线图、干系人 | 与 Memory 冲突 | 与 Memory ROOT 一致 |
| 文档索引 | 文档地图、缺口检查 | 所有人 | 文档列表、路径、读者、状态、关系 | 按角色分类 | 漏列关键文档、链接失效 | 覆盖 document-map 清单 |
| 术语表 | 统一术语 | 全员 | 术语-定义-备注 | — | 与 Memory 或实现不一致 | 被多文档引用且一致 |
| 系统架构 | 组件、边界、数据流、部署 | 架构、开发、运维 | 概述、边界、组件、数据流、部署拓扑、技术栈、非功能 | 容量假设、演进计划 | 只有图无文、与选型冲突 | 与 Memory/选型/部署一致 |
| 技术选型 | 选型理由与权衡 | 架构、开发 | 选型项、方案、理由、权衡与替代 | 版本号（可放配置文档） | 无理由、与架构冲突 | 可追溯决策 |
| 详细设计 | 模块、接口、逻辑 | 开发、测试 | 范围、与架构对应、模块划分、接口、数据流、关键逻辑 | 时序图、扩展点 | 大段贴代码、与 API 矛盾 | 可实现、与 API/DB 一致 |
| 环境说明 | 多环境差异 | 开发、运维 | 环境列表、差异、用途 | 网络与权限 | 与部署/配置矛盾 | 部署可依此执行 |
| 配置说明 | 配置项与安全 | 开发、运维 | 配置项、默认值、安全注意、引用环境 | — | 明文密钥、与实现不符 | 与代码/部署一致 |
| 部署手册 | 可重复部署与回滚 | 运维、发布 | 前置、拓扑、步骤、配置、验证、回滚、版本 | 蓝绿/金丝雀 | 步骤不可执行、无回滚 | 新人可独立部署与回滚 |
| 操作/运维 | 日常运维规范 | 运维 | 组件、日常操作、配置变更、监控入口、故障入口 | 值班与升级 | 与部署/架构不一致 | 可执行、有应急入口 |
| 监控与告警 | 指标与告警规则 | 运维 | 指标、采集、告警规则、级别、处理建议 | — | 规则与现网不符、无处理指引 | 告警有对应 Runbook |
| 接口/API | 调用约定与示例 | 开发、测试、对接方 | 基础信息、每接口路径/参数/响应/错误、错误码、示例、变更 | — | 与实现不一致、无错误码 | 可仅凭文档联调 |
| 数据库设计 | 表结构、索引、迁移 | 开发、DBA | 表清单、字段、索引、迁移约定、与设计对应 | ER 图 | 与详细设计/实现矛盾 | 与代码一致 |
| 测试计划 | 范围、策略、资源 | 测试、PM | 目标与范围、类型、环境、通过准则、风险 | — | 无通过准则、与环境脱节 | 可执行测试 |
| 验收/UAT | 可验证验收条件 | 产品、测试、客户 | 可验证条款、与需求对应、优先级 | — | 模糊描述 | 每条可判通过/不通过 |
| 安全设计 | 威胁与控制措施 | 架构、安全、审计 | 目标、假设、威胁与缓解、依赖与合规 | — | 与架构矛盾、泄露敏感 | 与实现与合规一致 |
| 权限模型 | 角色与权限 | 开发、安全 | 角色、权限矩阵、鉴权方式、配置衔接 | — | 与实现不一致 | 可配置与审计 |
| Runbook | 单故障处置 | 运维、值班 | 现象、原因、诊断、处理、升级、相关文档 | — | 步骤模糊、无升级 | 可按步骤执行 |
| ADR | 架构决策记录 | 架构、开发 | 标题、状态、背景、决策、选项、后果、元数据 | — | 无背景或选项、与现状矛盾未标 superseded | 可追溯决策 |
| Release Notes | 版本变更说明 | 全员、客户 | 版本、概述、新增/变更/修复/安全、升级与兼容 | 依赖与文档 | 与部署/实际发布不一致 | 与部署与版本一致 |
| FAQ | 常见问题 | 全员、支持 | 问题-答案、链接相关文档 | — | 过时答案 | 可解决常见问题 |
| Onboarding | 新成员上手 | 新成员 | 环境搭建、权限、首次运行、文档索引 | 文化/流程 | 步骤不可执行 | 新人可完成首次运行 |
| AI：模型与推理 | 模型、推理、配额 | 架构、算法、运维 | 模型与版本、推理方式、配额与限流、降级 | — | 与 Memory 冲突 | 与架构一致 |
| AI：Prompt 设计 | 提示词版本与安全 | 算法、产品、安全 | 用途与版本、原则、敏感信息处理、变更流程 | — | 全文贴敏感提示词 | 可评审与合规 |
| AI：评测方案 | 质量与回归 | 算法、质量 | 指标、数据集、执行方式、通过阈值 | — | 无通过阈值 | 可复现与门禁 |

## 文档治理机制摘要

- **盘点规则**：按 document-map 定期扫描 docs/ 与约定路径，列出已有文档及最后修改时间。
- **缺失识别**：对照 document-map 与项目类型（常规/AI），标出缺失与仅占位/过旧文档；依赖关系决定生成顺序。
- **新旧/版本识别**：依据元数据中的版本、日期、状态；与 Release Notes 比对。
- **冲突检测**：与 Memory ROOT、术语表、ADR、同主题文档比对；冲突时提示并建议（以 Memory 为准/以用户为准/更新 Memory）。
- **交叉引用规则**：相对路径或稳定 ID；生成时写入「相关文档」；同步时校验链接有效。
- **术语统一**：以术语表为权威；生成/更新时优先用术语表表述，新术语先入表再用。
- **状态标记**：draft → reviewed → approved；deprecated 时注明替代或 superseded by。
- **更新触发**：需求变更→验收/测试/PRD；架构变更→架构/设计/部署/运维；接口变更→API/调用方说明；发布→Release Notes/部署版本；事故/复盘→Runbook/监控/运维。

## 质量校验清单使用方式

- 单份文档：用 quality-checklist.md 逐项打勾。
- 全量审计：先盘点，再对关键文档（README、架构、部署、API、Runbook）执行清单；输出不通过项与修复建议。
- 发布前：执行 doc-release-check 逻辑 + 清单中与发布相关的项。

## 落地建议

1. **首次引入**：在项目中加入本 Skill 或 references/templates；指定 Memory ROOT 路径；运行 /doc-gap-analysis 得到缺口报告与优先级。
2. **补齐基础**：先生成文档索引、术语表、项目概述（与 Memory 对齐）；再按依赖顺序生成架构、环境、配置、部署、API、运维、Runbook 等。
3. **日常维护**：需求/架构/接口/发布/事故后调用 /doc-update 或人工按「更新触发」规则更新；定期 /doc-sync 统一术语与引用；发布前 /doc-release-check。
4. **与 APM 结合**：若使用 agentic-project-management，将 Memory ROOT 视为 APM Memory_Root.md；Phase 完成后在 Memory 中写摘要，并触发相关文档更新（部署、Release Notes、Runbook 等）。
5. **演进**：根据项目反馈增删 document-map 中的文档类型、调整各 guide 的必备章节、增加模板或命令。
