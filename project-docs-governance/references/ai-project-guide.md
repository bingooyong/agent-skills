# AI 项目专项文档指南

适用于含 LLM/模型、Agent、Prompt、评测等组件的项目。在通用文档体系基础上补充以下类型。

## 1. 模型与推理架构说明

**目的**：模型选型、推理服务形态、配额与限流、与业务组件的边界。  
**读者**：架构、算法、运维。

**必备**：使用的模型及版本、推理方式（API/自托管）、配额与限流策略、降级与多模型策略（若适用）。与 Memory ROOT 中的技术栈一致。

---

## 2. Prompt / System Prompt 设计说明

**目的**：提示词版本管理、评审与合规、敏感信息处理。  
**读者**：算法、产品、安全。

**必备**：Prompt 的用途与版本、设计原则、输入输出约定、敏感信息与 PII 处理、变更与评审流程。不强制把全文贴入文档，可引用仓库路径或配置。

---

## 3. Tool / MCP / 外部依赖说明

**目的**：工具列表、权限、限流与故障影响。  
**读者**：开发、运维、安全。

**必备**：调用的外部 API、MCP 或工具列表、权限与鉴权、限流与熔断、与 Runbook 的衔接。

---

## 4. Evaluation / Benchmark 方案

**目的**：质量与回归评估的指标、数据集与方法。  
**读者**：算法、质量、PM。

**必备**：评估指标、数据集或场景、执行方式与频率、通过阈值、与发布门禁的关系（若适用）。

---

## 5. Guardrails / 安全策略

**目的**：内容过滤、滥用防护、合规策略。  
**读者**：安全、产品、运维。

**必备**：策略目标、检测与拦截点、配置与开关、与监控/告警的衔接。

---

## 6. Token / Cost 管理说明

**目的**：用量、预算与成本控制。  
**读者**：架构、运维、成本负责人。

**必备**：计费维度、用量观测方式、预算与告警、优化建议（可选）。

---

## 7. 记忆 / 上下文管理说明

**目的**：上下文窗口、Memory ROOT、会话与长期记忆策略。  
**读者**：架构、开发、产品。

**必备**：上下文来源（Memory ROOT、会话、知识库）、更新与失效策略、与文档治理的衔接（如「文档与 Memory 一致」）。

---

以上文档若与通用文档（如架构、部署、运维）有重叠，可在通用文档中设「AI 专项」章节并链接到本类文档，避免重复与冲突。
