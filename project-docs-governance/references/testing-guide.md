# 测试文档指南

## 1. 测试计划

**目的**：界定测试范围、策略、资源与进度。  
**读者**：测试、PM、开发。

**必备**：测试目标与范围、测试类型（单元/集成/系统/UAT）、环境与数据、通过准则、风险与假设、与需求/验收的对应。  
**禁忌**：范围不清、无通过准则、与环境或需求文档脱节。

---

## 2. 验收标准 / UAT 文档

**目的**：可验证的验收条件，便于 UAT 与交付判断。  
**读者**：产品、测试、客户。

**必备**：每条为「可执行+可判断通过/不通过」的陈述；与需求/PRD 对应；可带优先级。  
**禁忌**：模糊描述（如「性能要好」）；与 Memory ROOT 或 PRD 目标不一致。

---

## 3. 测试用例设计说明（可选）

**目的**：说明用例设计思路、覆盖维度、数据与边界。  
**读者**：测试、开发。  
可引用测试计划与需求；与用例库或自动化脚本对应。
