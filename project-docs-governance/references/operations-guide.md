# 操作/运维手册与 Runbook 指南

## 1. 操作/运维手册

**目的**：日常运维动作的规范说明，便于值班与交接。  
**读者**：运维、SRE、值班。

**必备章节**：
- 服务/组件清单与职责（与架构一致）
- 日常操作：启停、扩缩容、备份、日志与轮转
- 配置变更流程（谁改、如何审批、如何回滚）
- 监控与告警入口（链接监控文档）
- 故障与应急入口（链接 Runbook 或故障处理文档）
- 值班与升级策略（可选）
- 相关文档：部署、监控、Runbook、安全

**禁忌**：与部署或架构不一致；步骤不可执行；缺「出问题找谁/找哪份文档」。

---

## 2. 监控与告警手册

**必备**：监控目标（指标列表）、采集方式、告警规则与阈值、告警级别、处理建议或链接 Runbook、值班与升级。  
**禁忌**：规则与现网不一致；告警无对应处理指引。

---

## 3. 故障处理 / Runbook

**目的**：针对具体故障现象给出诊断步骤与处理动作，减少 MTTR。  
**读者**：运维、值班。

**每则 Runbook 建议包含**：
- 故障现象/告警名称
- 可能原因（简短）
- 诊断步骤（命令、检查点、预期）
- 处理步骤（可执行）
- 若无法解决：升级对象、临时缓解措施
- 相关文档与变更记录

**禁忌**：步骤模糊无法执行；与监控/部署文档矛盾；无升级或回滚指引。
