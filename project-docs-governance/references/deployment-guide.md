# 部署手册指南

## 文档目的

让运维或自动化流水线能够可重复、可验证地完成部署与回滚。

## 目标读者

运维、发布负责人、CI/CD 维护者。

## 必备章节

1. **前置条件**  
   环境要求、权限、依赖服务、配置/密钥准备；引用环境说明与配置说明。

2. **部署拓扑与节点**  
   与架构一致；列出要部署的组件及目标环境（开发/测试/生产）。

3. **部署步骤（可执行）**  
   按顺序：拉取/构建、配置注入、数据库/迁移、服务启动、健康检查。每步可验证（命令 + 预期结果）。

4. **配置与密钥**  
   关键配置项说明或引用配置文档；密钥/凭证的获取与放置方式（不写明文）。

5. **验证与冒烟**  
   部署后如何确认成功（健康接口、关键流程检查）。

6. **回滚步骤**  
   触发条件、回滚顺序、数据/迁移的回滚策略（若适用）。

7. **版本与发布说明**  
   当前文档对应的版本；与 Release Notes 的对应关系。

## 可选

- 蓝绿/金丝雀说明
- 依赖服务启动顺序与超时
- 常见部署失败原因与排查

## 不应出现的问题

- 步骤缺命令或不可执行。
- 与架构/环境文档中的拓扑或环境名不一致。
- 缺少回滚或「出问题怎么办」的指引。
- 敏感信息明文（密钥、账号）。

## 质量判断标准

- 新人按文档可独立完成一次部署与回滚。
- 与 Memory ROOT、架构、环境说明无冲突。
