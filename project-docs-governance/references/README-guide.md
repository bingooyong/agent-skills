# README 与项目入口文档指南

## 文档目的

- 项目入口：任何人打开仓库首先看到的说明。
- 传达：项目是什么、解决什么问题、如何快速运行/使用、文档与资源在哪。

## 目标读者

所有人（开发者、运维、产品、外部合作方、新成员）。

## 必备章节

1. **项目名称与一句话描述**  
   与 Memory ROOT 中的项目概述一致。

2. **主要功能/特性**  
   3～5 条，可执行、可验证的表述。

3. **快速开始（Quick Start）**  
   最少步骤：环境要求、安装/构建、运行/试用、可选：首次使用示例。若步骤依赖环境或配置，链接到「环境说明」「配置说明」。

4. **文档索引**  
   指向文档地图或 `docs/README.md`，便于读者按角色找到详细文档。

5. **版本与许可**  
   当前版本或「见 Release Notes」；许可证类型。

6. **贡献/反馈（可选）**  
   指向 CONTRIBUTING、Issues、Code of Conduct 等。

## 可选章节

- 架构概览（一两句 + 链接到架构文档）
- 技术栈列表（或链接技术选型）
- 常见问题（或链接 FAQ）
- 路线图/状态（若需要）

## 不应出现的问题

- 与 Memory ROOT 或项目概述矛盾的目标/范围描述。
- 只有「详见某某文档」而无一句实质性介绍。
- 快速开始步骤缺环境/依赖说明或无法一次跑通。
- 过长的安装/构建步骤塞满 README（应拆到部署/开发文档）。

## 质量判断标准

- 新成员能否在 5 分钟内理解项目并完成「跑起来」。
- 术语与术语表、Memory ROOT 一致。
- 链接有效，文档索引存在且覆盖主要文档类型。
