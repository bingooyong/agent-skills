# 文档质量校验清单

用于单份文档或全量文档审计时的检查项。执行时优先依据 Memory ROOT 与术语表。

## 1. 元数据与状态

- [ ] 文档含版本或日期、状态（draft/reviewed/approved/deprecated）、维护者/作者、最后更新时间。
- [ ] 若有变更记录或变更摘要，与发布说明/CHANGELOG 无矛盾。
- [ ] 若标记为 deprecated，已注明替代文档或 superseded by。

## 2. 与 Memory ROOT 一致性

- [ ] 项目目标、范围、阶段描述与 Memory ROOT 一致或明确标注差异原因。
- [ ] 技术栈、架构决策与 Memory ROOT 或 ADR 无冲突。
- [ ] 术语与项目术语表一致；新术语已加入术语表再使用。

## 3. 目标读者与深度

- [ ] 文档明确或可推断目标读者。
- [ ] 深度与读者匹配（如对外文档不暴露内部预案细节）。
- [ ] 对外/对内边界清晰（若同一主题有内外两版）。

## 4. 可执行与可验证

- [ ] 操作类文档（部署、运维、Runbook）步骤可执行，含命令或操作说明及预期结果。
- [ ] 验收/测试类条款可判断通过/不通过。
- [ ] 接口文档与实现一致（或标注差异与计划修复）。

## 5. 交叉引用与依赖

- [ ] 引用的其他文档存在且路径正确。
- [ ] 上游文档（如架构之于详细设计）已存在或已在本文档中标记「待补充」。
- [ ] 无悬空引用（链接不失效）。

## 6. 冲突与过时

- [ ] 与同主题其他文档无事实性冲突；若冲突已标注并给出处理建议。
- [ ] 版本号、发布日期与 Release Notes 一致。
- [ ] 无明显过时描述（如已下线功能仍写为推荐）。

## 7. 安全与合规

- [ ] 无明文密钥、账号、内部漏洞细节。
- [ ] 对外文档已脱敏；合规相关表述准确（若适用）。

## 8. 文档治理规则摘要

- **盘点**：按 document-map 定期盘点，识别缺失与过旧。
- **冲突检测**：与 Memory ROOT、术语表、ADR 及其他文档比对。
- **状态流转**：draft → reviewed → approved；deprecated 时注明替代。
- **更新触发**：需求/架构/接口/发布/事故/复盘后，触发对应文档更新并执行本清单抽查。
