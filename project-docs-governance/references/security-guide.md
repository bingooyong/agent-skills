# 安全与权限文档指南

## 1. 安全设计说明

**目的**：说明安全目标、威胁假设、控制措施与合规要点。  
**读者**：架构、安全、审计、运维。

**必备**：安全目标与范围、假设与信任边界、主要威胁与缓解（认证、授权、数据保护、审计日志）、依赖与供应链注意、合规要点（若适用）。  
**禁忌**：与架构或实际实现矛盾；敏感细节（密钥、内部漏洞）写入对外文档。

---

## 2. 权限模型说明

**目的**：角色、权限、鉴权方式与配置方式。  
**读者**：开发、安全、运维。

**必备**：角色定义、权限列表或矩阵、鉴权方式（RBAC/ABAC 等）、与配置/部署的衔接。  
**禁忌**：与实现不一致；缺「如何申请/变更权限」的流程。
