# 接口/API 文档指南

## 文档目的

让调用方（前端、合作方、测试）能正确调用接口：路径、方法、参数、响应、错误码与示例。

## 目标读者

开发、测试、对接方、自动化脚本作者。

## 必备内容

1. **基础信息**  
   基础 URL、认证方式（如 Bearer、API Key）、通用请求头、版本策略。

2. **每个接口**  
   - 路径、HTTP 方法
   - 请求：路径/查询/体参数、类型、必填、说明
   - 响应：成功体结构、状态码；错误体结构（code、message、details）
   - 示例请求与响应（可精简）
   - 与详细设计/数据模型的对应（可引用）

3. **错误码统一说明**  
   通用错误码表及处理建议。

4. **变更与版本**  
   与发布节奏一致；破坏性变更标注并指向 Release Notes。

## 形式建议

- OpenAPI/Swagger 与 Markdown 互补：机器可读用 OpenAPI，人类阅读用 Markdown 或由 OpenAPI 生成。
- 若项目已有 OpenAPI，Markdown 可只保留概述、认证、示例与变更说明，细节以 OpenAPI 为准。

## 不应出现的问题

- 与实现不一致的路径/参数/响应。
- 缺错误码或错误体结构。
- 无示例或示例无法直接复用。
- 与详细设计/术语表术语不一致。

## 质量判断标准

- 测试或调用方能仅凭文档完成联调。
- 与 Memory ROOT、详细设计、术语表一致。
